# AGENTS.md - Code Reader 项目开发规范

## 文档更新规则

**每次代码变更后，必须同步更新以下文档：**

1. **`docs/CHANGELOG.md`** — 记录变更内容（新增/修复/改进），按版本号分组
2. **`docs/TEST_REPORT.md`** — 如果涉及 bug 修复，将对应条目状态更新为已修复

### 具体流程

- 修复一个 bug → 在 CHANGELOG 的当前版本下添加 `### 修复` 条目
- 新增功能 → 在 CHANGELOG 的当前版本下添加 `### 新增` 条目
- 改进优化 → 在 CHANGELOG 的当前版本下添加 `### 改进` 条目
- 测试报告中的 bug 被修复 → 将对应条目标记为 `✅ 已修复 (日期)`
- 每次修改后 → git commit 并 push

### 版本号规则

- 修复 bug → 补丁版本号 +1（v0.1.0 → v0.1.1）
- 新增功能 → 次版本号 +1（v0.1.0 → v0.2.0）
- 重大变更 → 主版本号 +1（v0.1.0 → v1.0.0）
