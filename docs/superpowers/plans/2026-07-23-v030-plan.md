# v0.3.0 tree-sitter 语法高亮 + 代码折叠 + 符号列表 — 实现计划

> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 集成 tree-sitter 到 C++ native addon，实现语法高亮着色、代码折叠交互、符号列表导航。

**Architecture:** C++ 层新增 grammar_registry 并重写 parser 接入 tree-sitter；N-API 导出 async parseFile；Webview reader.ts 渲染 token 着色和折叠按钮；TS 新增 SymbolTreeProvider。

**Tech Stack:** C++17 + N-API + tree-sitter, libcurl(ollama预留), TypeScript + VSCode API

## Task 3: parser 重写 + SymbolExtractor + FoldingProvider (C++)

**目标:** 实现真正的 tree-sitter 解析流程，提取 tokens、折叠范围、符号。

**Files:** Modify `parser.cpp`, `symbolExtractor.cpp`, `foldingProvider.cpp`

- [ ] **3-1** — 在 `parser.cpp` 中为 `Parser::Impl` 添加 `GrammarRegistry` 成员和互斥锁
- [ ] **3-2** — 实现 `vscodeLangToTreeSitter()` 函数: language id → tree-sitter 语言名映射
- [ ] **3-3** — 实现 `nodeTypeToTokenType()` 函数: tree-sitter node_type + parent_type → "keyword"/"string"/"comment"/"number"/"function"/"class"/"text"
- [ ] **3-4** — 重写 `parse(content, language)`: 创建 TSParser → 解析 → 递归遍历 AST: 叶子节点提取 Token(含 start_byte/end_byte/line/type)，多行父节点提取 FoldRange → 调用 SymbolExtractor → 返回 ParseResult
- [ ] **3-5** — 实现 `SymbolExtractor::extract()`: 遍历 AST 顶层声明节点(function_declaration/class_declaration/method_definition/variable_declaration)，提取 name、kind、start_line/end_line，递归 children
- [ ] **3-6** — 实现 `FoldingProvider::compute()`: 收集所有多行节点为 FoldRange 列表，排除 import 块(默认折叠)和根节点
- [ ] **3-7** — 编译验证: `npm run build:native`
