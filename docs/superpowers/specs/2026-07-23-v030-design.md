# Code Reader v0.3.0 — 设计规约

> 日期: 2026-07-23 | 状态: 已批准 | 上接 v0.2.0

## 1. 概述

v0.3.0 将 tree-sitter 集成到 C++ native addon，实现真实的代码解析，使阅读面板获得语法高亮着色、代码折叠交互，侧边栏新增符号列表导航。

## 2. 范围

| 功能 | v0.3.0 | v0.3.1 |
|------|--------|--------|
| tree-sitter 集成 (build + grammar registry) | ✅ | - |
| 语法高亮着色 (keyword/string/comment/number/function/class) | ✅ | - |
| 代码折叠 (AST 驱动的折叠范围 + Webview 按钮) | ✅ | - |
| 符号列表 (侧边栏 TreeView, 点击跳转) | ✅ | - |
| 定义跳转 | ❌ | ✅ |
| 引用查找 | ❌ | ✅ |
| 调用关系图 | ❌ | ✅ |

## 3. 架构变更

v0.2.0 C++ 层为空壳。v0.3.0 让 C++ parser 真正工作：

```
Before: C++ parser → 空壳, 返回原始内容
After:  C++ parser → tree-sitter AST → tokens + folds + symbols
```

**涉及模块：**

| 层 | 文件 | 操作 | 职责 |
|-----|------|------|------|
| C++ | `binding.gyp` | 修改 | 链接 tree-sitter 库和 grammar 源文件 |
| C++ | `grammar_registry.h/.cpp` | 新增 | 管理 TS/JS/Python/C/C++ 语言 grammar |
| C++ | `parser.cpp` | 重写 | tree-sitter AST 遍历, token/fold/symbol 提取 |
| C++ | `addon.cpp` | 修改 | N-API 异步导出 parseFile |
| TS | `reader.ts` | 修改 | token 字节偏移 → `<span class="token-xxx">` 着色 + 折叠按钮 |
| TS | `symbolTreeProvider.ts` | 新增 | 符号列表 TreeView, 点击跳转行 |
| TS | `readerPanel.ts` | 修改 | 将 symbols 发送给 SymbolTreeProvider |
| TS | `package.json` | 修改 | 注册 symbol 视图到侧边栏 |

## 4. 数据流

```
openReader(filePath)
  → readFileContent()
  → C++ parseFile(content, language) [AsyncWorker]
    → GrammarRegistry → TSParser → AST traversal
    → tokens[] + folds[] + symbols[]
  → webview.postMessage({ loadFile, result })
    → reader.ts: 着色 + 折叠
  → symbolTreeProvider.update(symbols)
    → 侧边栏树形渲染, 点击跳转
```

## 5. 关键决策

- **语言支持**：v0.3.0 注册 TypeScript/JavaScript/Python/C/C++ 五种 grammar
- **N-API 异步**：parseFile 包装为 `Napi::AsyncWorker`，不阻塞 UI 线程
- **降级策略**：不支持的 language 返回空数组，Webview 降级纯文本；原生模块不可用 catch 降级
- **Token 类型映射**：树叶节点的 `node_type` → 字符串 (`keyword`/`string`/`comment`/`number`/`function`/`class`/`text`)，Webview 侧 CSS class `token-xxx`
